# Version Control Notes

## 1. What is a Version Control System (VCS)?
- A VCS is a system that tracks changes in files over time.
- It helps store different versions of source code.
- Users can review, compare, and restore previous versions.
- Common VCS tools include Git, SVN, and Mercurial.

## 2. Why do we need to use VCS?
- To avoid losing source code when errors occur.
- To support teamwork on the same project.
- To track who changed what and when.
- To manage multiple versions of a project safely.

## 3. Advantages and Disadvantages of VCS

### Advantages:
- Keeps full history of file changes.
- Supports collaboration among team members.
- Allows rollback to previous versions.
- Improves project management and productivity.

### Disadvantages:
- Takes time to learn and use correctly.
- Conflicts may occur when merging changes.
- Requires following proper workflows.

## 4. Create a new repository and first commit
- Initialize a Git repository.
- Add files to the staging area.
- Commit changes with a meaningful message.
- Store project history in the repository.
